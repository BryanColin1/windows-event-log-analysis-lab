# Failed Logon Event Analysis

## Objective

Analyze failed Windows logon events and explain their security relevance for SOC monitoring.

## Background

Failed logon events can indicate mistyped credentials, unauthorized access attempts, brute-force activity, or misconfigured services. In Windows Security logs, failed logons are commonly associated with Event ID `4625`.

## Test Activity

A failed logon event can be generated by intentionally entering an incorrect password for a local Windows account in a controlled lab environment.

## Analysis Questions

| Question | Purpose |
|---|---|
| Which account experienced the failed logon? | Identifies the targeted account |
| What time did the event occur? | Supports timeline analysis |
| What logon type was recorded? | Helps identify local, remote, or network logon |
| What was the failure reason? | Helps determine cause |
| Was the activity repeated? | Helps identify brute-force behavior |

## Security Relevance

Failed logon events are important for detecting unauthorized access attempts, brute-force activity, password guessing, and suspicious authentication behavior.

## Recommended Response

1. Identify the affected account.
2. Review the timestamp and source information.
3. Check whether multiple failed attempts occurred.
4. Determine whether the activity was expected or suspicious.
5. Escalate if repeated failures or unusual source systems are observed.
